# Skill: Presentation Deck (PPTX)

## Purpose
Produce executive-ready PowerPoint decks for any organizational context: governance (ARB), business strategy, technology briefings, stakeholder communications, or decision support.

## When to use
- Executive presentations
- ARB/governance submissions
- Business strategy communication
- Technology briefings and demos
- Stakeholder alignment meetings
- Board-level communications

## Inputs
- Topic, architecture, strategy, or initiative
- Target audience (C-suite, technical, mixed)
- Key messages and narrative arc
- Optional: template constraints, tone, timeline
- Optional: existing artifacts to integrate

## Outputs
- Structured slide content
- PPTX-ready sections with speaker notes
- Visual hierarchy and flow guidance
- Accessibility annotations

## Constraints
- Minimal text per slide (max 5-6 lines)
- Strong visual-first thinking
- Narrative consistency across deck
- Audience-appropriate language and tone
- WCAG accessibility compliance

## Workflow
1. **Understand audience & goal** - Define decision/action expected
2. **Craft narrative arc** - Opening → Problem/Opportunity → Solution/Recommendation → Next Steps
3. **Map key messages** - 3-5 supporting points with evidence
4. **Design structure** - Title, section breaks, supporting slides
5. **Apply visual patterns** - Consistent color, typography, layout
6. **Embed specifics** - Data, diagrams, quotes, concrete examples
7. **Generate speaker notes** - Talking points for presenter
8. **Validate accessibility** - Color contrast, alt text, readable fonts

## Quality checks
- Clear story arc with obvious call-to-action
- Minimal, punchy text
- Each slide has purpose
- Visuals support narrative (not decorative)
- Terminology consistent with audience
- Speaker notes help presenter deliver
- Accessible to all viewers
- Timeable (60 sec per slide is reasonable default)

## Output formats
- PPTX structure (slide-by-slide JSON/YAML)
- HTML speaker notes
- Accessibility metadata (alt text, color values)
- Timing guide
